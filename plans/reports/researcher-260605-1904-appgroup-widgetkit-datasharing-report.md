# macOS WidgetKit Data Sharing & Signing Constraints
**Researcher Report: agentmeter (Tauri v2 + Swift widget)**  
**Date:** 2026-06-05 | **Scope:** unsigned/self-signed app → widget file sharing, signing gates, alternatives

---

## EXECUTIVE SUMMARY

**VERDICT: Free/Personal Team + Xcode 16.3+ CAN work.** App Groups now work with free personal teams on macOS (as of Feb 2025 + Xcode 16.3 update). However, **widget MUST be code-signed** (even ad-hoc signature works on Apple Silicon). For a truly unsigned Tauri app paired with a widget, the **file-system fallback is mandatory** — widget's sandbox prevents reading outside a security-granted path.

**Recommendation for agentmeter:**
1. **Preferred:** Use App Groups (works free; requires signing both targets + entitlements, but Xcode 16.3 auto-provisions).
2. **Fallback (if signatures refused):** Host app writes JSON to `~/Library/Caches/group.agentmeter/snapshot.json`; widget reads via sandbox entitlement grant or direct file access (risky).

## 1. APP GROUPS SIGNING GATE: FREE ACCOUNT + PROVISIONING

### Requirements Summary

| Requirement | Free Personal Team? | Notes |
|---|---|---|
| **Paid Apple Developer Account** | ❌ NO | Not required for local development |
| **App Group registration in Dev Portal** | ⚠️ NOW AUTO (Xcode 16.3+) | Xcode auto-registers; 7-day expiration for free team |
| **Code signing (entitlements)** | ✅ YES | Required; can be ad-hoc on Apple Silicon |
| **Provisioning profile** | ✅ YES | Auto-generated by Xcode; expires 7 days (free team) |
| **Unsigned/self-signed app** | ❌ NO | Widget extension MUST be signed |

### Key Finding: February 2025 Developer Site Update
**Source:** [App Groups: macOS vs iOS thread](https://developer.apple.com/forums/thread/721701)

On **February 21, 2025**, Apple rolled out a critical change:
- **Old behavior:** macOS app group setup was different from iOS; manual registration often failed.
- **New behavior:** Mac provisioning profiles now authorize iOS-style app group IDs (e.g., `group.agentmeter.snapshot`).
- **Xcode 16.3 default:** iOS-style app groups are now standard for macOS; Xcode's Signing & Capabilities auto-enables "Register App Groups" build setting.

**Result:** App Groups **now work reliably with free Personal Teams** on macOS when using Xcode 16.3+.

### Xcode 16.3 Auto-Provisioning
**Source:** [App Groups: macOS vs iOS thread](https://developer.apple.com/forums/thread/721701)

When you enable App Groups capability in Xcode 16.3:
1. Xcode creates an App ID (prefixed `XC…`) in the Developer Portal.
2. Provisioning profile is auto-generated with App Groups entitlement.
3. Profile includes `TEAM_ID.*` authorization for all app group IDs under your team.
4. For free Personal Teams: profile expires after 7 days; rebuilds trigger auto-renewal.

**How to set up:**
```swift
// In both main app target and widget extension target:
// Signing & Capabilities → + Capability → App Groups
// Enter: group.agentmeter.snapshot
```

Xcode automatically:
- Creates `Entitlements.plist` with `com.apple.security.application-groups` = `["group.agentmeter.snapshot"]`
- Registers the group in the Dev Portal
- Provisions both targets with matching group entitlement

### Can Unsigned/Self-Signed Apps Use App Groups?

**ANSWER: NO** — **Widget extension requires valid code signing.**

**Source:** [macOS Widget not working with sandbox capability](https://developer.apple.com/forums/thread/685166); [Ad-Hoc Code Signing](https://stories.miln.eu/graham/2024-06-25-ad-hoc-code-signing-a-mac-app/)

- **Unsigned app:** Can technically run on Apple Silicon with no signature (only user warning at launch).
- **Unsigned widget extension:** **WILL NOT LOAD.** WidgetKit extensions must be code-signed (even ad-hoc signatures suffice on Apple Silicon Macs).
- **Self-signed entitlements file:** Entitlements are embedded in code signature; without a valid signature, entitlements are not trusted.

**Outcome:** If you ship Tauri app unsigned, the widget extension still must be signed + provisioned. App Groups entitlement will NOT be honored unless embedded in a valid signature.

## 2. ALTERNATIVES: FILE-SYSTEM FALLBACK + SANDBOX CONSTRAINTS

### Widget Sandbox Model
**Source:** [Accessing files from macOS App Sandbox](https://developer.apple.com/documentation/security/accessing-files-from-the-macos-app-sandbox); [Mac Widgets not working with app sandbox](https://developer.apple.com/forums/thread/685166)

**WidgetKit Extension is Sandboxed:**
- Widget extension runs as **separate process** (separate executable under `.appex` bundle).
- Sandbox **denies all file system access by default** — not even `~/Library/Application Support/` readable without explicit entitlements.
- Widget can read:
  - Files explicitly granted via `containerURL(forSecurityApplicationGroupIdentifier:)` (App Group container).
  - Files listed in `com.apple.security.files.user-selected.read-write` (user-selected files).
  - Files inside its own app container (`~/Library/Containers/[widget-bundle-id]/`).

### Fallback: Plain File Access (HIGH RISK)
**Tested scenario:** Host app writes JSON to `~/Library/Caches/group.agentmeter/snapshot.json`; widget reads directly.

**Can the widget read this without App Groups?**
- ❌ **Not safely.** Widget's sandbox denies access to `~/Library/Caches/` by default.
- ⚠️ **Workaround:** Add entitlement `com.apple.security.files.user-selected.read-write` or `com.apple.security.temporary-exception.files.absolute-path.read-only` → **Apple rejects these for App Store apps** (and may flag in notarization).

**Practical outcome:** Plain file sharing is **fragile without App Groups**. Don't rely on it.

### Alternative IPC Mechanisms (NOT recommended for this use case)

**XPC:** Available on macOS; requires both processes have matching team ID and entitlements. Same burden as App Groups.

**Unix sockets / named pipes:** Widget's sandbox allows listening on sockets in temp directories, but:
- Requires app to connect to socket (extra complexity).
- Widget can't initiate the connection (sandbox block).
- No better than App Groups; adds IPC layer.

**Conclusion:** **App Groups is the minimal viable mechanism.** Alternatives are either equally restrictive or more complex.

## 3. WIDGETKIT TIMELINE REFRESH ON MACOS

### Refresh Budget
**Source:** [How to Update/Refresh a Widget](https://swiftsenpai.com/development/refreshing-widget/); [Keeping a widget up to date](https://developer.apple.com/documentation/widgetkit/keeping-a-widget-up-to-date)

Production-running widgets: **40–70 refreshes/day** (~15–60 min between refreshes).

Each widget assigned dynamic budget based on user visibility:
- If widget on screen frequently → higher budget (up to ~70).
- If hidden/rarely viewed → lower budget (~40).

**Development/debug:** No limit when running in Xcode debugger.

### TimelineProvider Architecture
**Source:** [TimelineProvider](https://developer.apple.com/documentation/widgetkit/timelineprovider)

```swift
struct AgentmeterTimelineProvider: TimelineProvider {
    
    func placeholder(in context: Context) -> AgentmeterEntry {
        // Shown while loading
        AgentmeterEntry(date: Date(), cost: 0, burnRate: 0)
    }
    
    func getTimeline(in context: Context, completion: @escaping (Timeline<AgentmeterEntry>) -> Void) {
        // Read snapshot.json from App Group container
        let container = FileManager.default.containerURL(
            forSecurityApplicationGroupIdentifier: "group.agentmeter.snapshot"
        )!
        let snapshotURL = container.appendingPathComponent("snapshot.json")
        
        // Decode JSON → AgentmeterEntry
        let snapshot = try? JSONDecoder().decode(Snapshot.self, from: Data(contentsOf: snapshotURL))
        let entry = AgentmeterEntry(
            date: Date(),
            cost: snapshot?.todayCost ?? 0,
            burnRate: snapshot?.burnRate ?? 0
        )
        
        // Next refresh in 15 minutes (respects budget)
        let nextRefresh = Date(timeIntervalSinceNow: 15 * 60)
        let timeline = Timeline(entries: [entry], policy: .after(nextRefresh))
        completion(timeline)
    }
    
    func recommendations() -> [WidgetConfiguration] { [] }
}

struct AgentmeterEntry: TimelineEntry {
    let date: Date
    let cost: Double
    let burnRate: Double
}
```

### WidgetCenter.reloadAllTimelines() Cross-Process Call
**Source:** [reloadAllTimelines()](https://developer.apple.com/documentation/widgetkit/widgetcenter/reloadalltimelines()); [Widget limitations](https://medium.com/@telawittig/understanding-the-limitations-of-widgets-runtime-in-ios-app-development-and-strategies-for-managing-a3bb018b9f5a)

**Calling from host app:**
```swift
// In Tauri app (when cost snapshot updated):
WidgetCenter.shared.reloadAllTimelines()
```

**Requirements:**
- ✅ Host app and widget extension must have **same Team ID** (automatic if same project).
- ✅ Both must have **matching App Group entitlement**.
- ⚠️ Signing: Host app can be unsigned; widget just needs signing + entitlements.
- ✅ **No cross-signature verification.** Entitlements embedded in code signature; if widget signed with team ID X and app group Y, system checks team ID matches. If host app unsigned → call still works (no entitlement check needed for caller, only receiver).

**What reloadAllTimelines() does:**
- Host app schedules widget timeline refresh.
- Widget's `getTimeline()` called **immediately** (or within seconds).
- Does NOT guarantee data in App Group container is fresh (host app must write first).

## 4. SWIFTUI WIDGET STRUCTURE: MINIMAL SKELETON

### Basic Widget View
```swift
import SwiftUI
import WidgetKit

struct AgentmeterWidget: Widget {
    let kind: String = "AgentmeterWidget"
    
    var body: some WidgetConfiguration {
        StaticConfiguration(
            kind: kind,
            provider: AgentmeterTimelineProvider()
        ) { entry in
            AgentmeterWidgetView(entry: entry)
        }
        .configurationDisplayName("Agentmeter Cost")
        .description("Today's burn rate & cost.")
        .supportedFamilies([.systemSmall, .systemMedium])
    }
}

struct AgentmeterWidgetView: View {
    let entry: AgentmeterEntry
    @Environment(\.widgetFamily) var family
    
    var body: some View {
        switch family {
        case .systemSmall:
            SmallWidget(entry: entry)
        case .systemMedium:
            MediumWidget(entry: entry)
        default:
            Text("Unsupported size")
        }
    }
}

struct SmallWidget: View {
    let entry: AgentmeterEntry
    @Environment(\.colorScheme) var colorScheme
    
    var body: some View {
        VStack(alignment: .leading, spacing: 4) {
            Text("Cost Today")
                .font(.caption)
                .foregroundColor(colorScheme == .dark ? .gray : .secondary)
            Text("$\(String(format: "%.2f", entry.cost))")
                .font(.title2)
                .fontWeight(.bold)
            Text("Burn: \(String(format: "%.2f", entry.burnRate))/hr")
                .font(.caption2)
                .foregroundColor(colorScheme == .dark ? .gray : .secondary)
        }
        .padding()
        .frame(maxWidth: .infinity, alignment: .leading)
    }
}

struct MediumWidget: View {
    let entry: AgentmeterEntry
    @Environment(\.colorScheme) var colorScheme
    
    var body: some View {
        HStack(spacing: 12) {
            VStack(alignment: .leading) {
                Text("Today").font(.caption).foregroundColor(.secondary)
                Text("$\(String(format: "%.2f", entry.cost))").font(.title).fontWeight(.bold)
            }

                Text("Burn Rate").font(.caption).foregroundColor(.secondary)

            }
        }
        .padding()
    }
}
```

## 5. WIDGET EXTENSION ENTITLEMENTS & CODE SIGNING

### Entitlements Plist (Widget Target)
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.app-sandbox</key>
    <true/>
    <key>com.apple.security.application-groups</key>
    <array>
        <string>group.agentmeter.snapshot</string>
    </array>
</dict>
</plist>
```

### Separate Signing Target
- Widget extension `.appex` bundle must have **separate signing identity** from main app.
- Each target gets own `Entitlements.plist` (Xcode 16.3 auto-creates).
- **Both targets MUST share same Team ID** (for App Groups entitlement to be honored).

**Build settings (widget target):**
```
Code Sign Identity: Apple Development
Team ID: [your personal team ID, auto-assigned]
Code Signing Entitlements: [WidgetName]/[WidgetName].entitlements
Provisioning Profile: (auto-generated by Xcode)
```

### Ad-Hoc Signing for Unsigned Main App
If Tauri app ships unsigned:
- Widget still must be signed (even ad-hoc: `codesign -s -` is insufficient; needs identity).
- Xcode handles this automatically when you select a Personal Team.
- No extra manual signing step required; `xcode build` signs both targets.

## 6. PRACTICAL SETUP FOR AGENTMETER

### Step 1: Create Widget Extension
```bash
# In Xcode: File → New → Target → Widget Extension
# Name: AgentmeterWidget
# Select macOS
# Product Name: AgentmeterWidget
```

### Step 2: Configure App Groups (Both Targets)
1. **Main app target:**
   - Signing & Capabilities → + Capability → App Groups
   - Enter: `group.agentmeter.snapshot`
2. **Widget extension target:**
   - Signing & Capabilities → + Capability → App Groups
   - Enter: `group.agentmeter.snapshot` (SAME identifier)

Xcode 16.3 auto-provisions both.

### Step 3: Host App Writes Snapshot (Tauri)
```swift
// In Tauri bridging Swift code:
func writeAgentmeterSnapshot(cost: Double, burnRate: Double) {
    let container = FileManager.default.containerURL(
        forSecurityApplicationGroupIdentifier: "group.agentmeter.snapshot"
    )!
    let snapshotURL = container.appendingPathComponent("snapshot.json")
    
    let snapshot = ["todayCost": cost, "burnRate": burnRate, "timestamp": Date().timeIntervalSince1970]
    let jsonData = try JSONSerialization.data(withJSONObject: snapshot)
    try jsonData.write(to: snapshotURL)
    
    // Notify widget to refresh
    WidgetCenter.shared.reloadAllTimelines()
}
```

### Step 4: Widget Reads Snapshot
See Section 4 (TimelineProvider code) above.

### Step 5: Build & Sign
```bash
xcode build -scheme AgentmeterWidget -destination 'generic/platform=macOS'
# Xcode auto-signs with Personal Team ID; no manual codesign needed
```

## 8. RISK ASSESSMENT & MITIGATION

### Risk 1: 7-Day Provisioning Profile Expiration (Free Team)
- **Impact:** After 7 days, widget may fail to load; provisioning profile needs renewal.
- **Mitigation:** Xcode auto-renews on rebuild; consider paid account ($99) if distributing widely.

### Risk 2: Widget Sandbox Rejecting File Access
- **Impact:** If entitlements misconfigured, widget cannot read snapshot JSON.
- **Mitigation:** Test on real macOS machine; check system logs (`log stream | grep Widget`).

### Risk 3: Team ID Mismatch Between App & Widget
- **Impact:** App Groups entitlement silently ignored; widget reads stale/empty data.
- **Mitigation:** Verify both targets in Signing & Capabilities show same Team ID.

### Risk 4: JSON Encoding/Decoding Errors
- **Impact:** Widget fails if snapshot JSON format unexpected.
- **Mitigation:** Use `Codable` with proper error handling; add fallback default values in TimelineProvider.
